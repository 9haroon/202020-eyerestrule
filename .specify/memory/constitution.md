# Project Standards

## Code Quality

Prioritise speed of delivery. Technical debt is acceptable at this stage — document shortcuts with TODO comments for future cleanup.

## Testing Requirements

**Minimal** — no automated tests required

## Error Handling

Basic error boundaries to prevent full-page crashes. Errors logged to console.

## Performance Constraints

No performance constraints for this project.

## UX Consistency

Functional first. Consistency can be refined once the concept is validated.
